# SocraticNovel — 三份文档的制作计划

## 项目定位
**SocraticNovel** = 苏格拉底小说。一个开源框架，让任何人都能创建自己的"二次元老师 + 轻小说叙事 + 苏格拉底教学"沉浸式学习系统。

与 AnimaTutor 无关，是独立项目。AP Physics EM 系统是 SocraticNovel 的第一个实战案例。

## 约束
- 中英双语
- 支持 Claude Code（首选）和知识库平台（Claude Projects / ChatGPT / Kimi）
- 支持任何学科，1-3 位自定义老师
- AP Physics EM 作为实战演示案例

## 文件三：META_PROMPT.md — 创造主引擎

**读者**：想创建自己的 SocraticNovel 系统的人（通过 AI 辅助）
**风格**：这是一个给 AI 读的 prompt，但人类需要能理解它在做什么
**做法**：最后写（在前两份文档写完后，充分理解了系统的所有层次）

### 设计思路
Meta Prompt 是一个**交互式生成器**。用户把它喂给 AI（Claude Code 或其他），AI 通过一系列提问来收集信息，然后生成完整的文件系统。

### 结构
1. **YAML 头部** — 用户填写基本信息：
   - 学科名称 + 教材信息
   - 老师数量（1-3）
   - 目标平台（Claude Code / 知识库）
   - 叙事风格偏好（轻小说 / 现实主义 / 科幻 / 自定义）
2. **Phase 1: 角色创建** — AI 提问环节：
   - 每个老师的名字、性别、年龄
   - 性格关键词（AI 会追问细化）
   - 教学风格偏好
   - 与学习者的关系基调
   - 核心矛盾/暗线（AI 会引导）
3. **Phase 2: 世界观构建** — AI 提问环节：
   - 故事发生在哪里？
   - 什么让这个地方特殊？
   - 学习者为什么来到这里？
   - 角色之间的关系（相互认识？）
4. **Phase 3: 课程结构** — AI 提问环节：
   - 有多少章节/课次？
   - 知识点如何分布？
   - 练习题从哪来？
5. **Phase 4: 故事线设计** — AI 提问环节：
   - 几个情感阶段？
   - 每个阶段的情感基调？
   - 有哪些必然事件？
   - 有哪些暗线种子？
6. **Phase 5: 生成** — AI 生成完整文件系统：
   - CLAUDE.md / BOOT.md
   - system.md
   - story.md
   - story_progression.md
   - characters/*.md
   - curriculum.md
   - runtime/*.md 模板
   - （如果是知识库模式：合并为 8+1 文件）
7. **附录：AP Physics EM 案例** — 作为"如果你不知道怎么回答 AI 的问题，看看这个例子"的参照

### 核心原则
- AI 必须不断提问，不能自作主张
- 每次生成一个模块后让用户 review
- 写作规则（散文标准、情绪工具箱等）自动继承，不需要用户知道
- 但故事设计（角色、世界观、情感阶段）完全由用户决定
