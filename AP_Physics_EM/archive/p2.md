# AI系统学习体验小结

> **非常感谢吴老师如此细致地介绍了这么好玩的学习体系。**

![](https://pica.zhimg.com/v2-0759414298fa71ea380c06795ff598ca.jpg?source=7e7ef6e2&needBackground=1)[怎样用AI让自己沉迷学习？502 赞同 · 98 评论 ](https://zhuanlan.zhihu.com/p/2012398047620014256)​[文章]()

周三看到吴老师[这篇文章](https://zhuanlan.zhihu.com/p/2012398047620014256)，扫了两眼，感觉很有趣，但写得挺长，看到后面感觉有点复杂，打算周末再仔细看看研究一下。然而没过几个小时忍不住好奇又点了开来，这次看到吴老师说可以把文章直接丢给 AI 来定制类似的系统（不知道第一次看没注意到是因为太过粗心😆 还是这个提示开始不在第一节），于是等不及仔细看完决定马上用 [cursor] 试一下。

我对经济学有点兴趣，但之前基本只限于看过一些科普；几次尝试公开课或者教材，都是学了几小时没坚持下去。于是这次打算拿经济学原理试试水。

到目前为止上了四节课，综合体感是在有一定心理预期的前提下仍然超过预期。

为什么这么说？我第一天上完课，竟然发现自己腰酸背痛，原来是学得太入神脖子前伸了。这个现象除了偶尔写代码 debug 上头偶尔会出现，基本都是打 [RTS 游戏]才会有的副作用。

到了第二天学习的时候新鲜感终于有点过去了，但感觉还是很不错。

正像吴老师所说，新知识的引入都是从问题开始，非常自然，有种回到本科跟同学吃饭时谈天说地的感觉。

而且我也是会经常问东问西，有时不免脱离教材（这个在第四节供需的课上比较明显）。以前学生阶段找老师/朋友这么提问，有时候人会不耐烦反问我，你问这个干什么？但 AI 老师（或者说朋友）就不一样，既有耐心也不 judge，而且因为知识面广，很多教材以外的问题也能答上来，这是真人老师未必能做到的。

另外一个优点是，这个系统可以扔进去英文教材但用中文授课。适合我这种比较作的，看英文教材嫌慢、但中译本又嫌质量差。

此外，正如原文评论区里翁律师所说，文字的效率是非常高的。看视频学习总是让我不耐烦，所以一般都是两倍速甚至三倍速播放，而且碰到不太理解的地方会很头疼，每次都要回跳一小段重听，但每次都很难精准定位回跳的位置。

这种文字学习的方式，不免让我想起大三在一个叫 [dataquest] 的网站上学 [python] 的经历。网站的设计是左边文字讲解，右边是编程环境。看完讲解之后可以直接写代码实践，即时反馈。这是不是跟现在这套 ai 学习体系非常相似呢？而当时学 python 的过程可能是我学习编程过程中最高效且愉悦的一段经历，似乎不是巧合。（虽然不太确定当时花了不少时间学习 matplotlib 是不是有意义😆）

我以前做家教带高中生和亲戚的时候，也尝试过类似苏格拉底教学的方式。当学生碰到不会的题，我会把让学生卡住的知识点拿过来想出一道新题，让学生继续尝试。如果新题也卡住了，再用卡点出一道题，如此循环直到定位到学生的底层知识缺陷，等学生掌握以后再逐步回到原题。应该说，这种递归式的教学解决知识点漏洞是挺高效的，不在学生掌握的知识点上反复练习浪费时间。然而，从实践经验来看对学生心理打击有点大——因为是反复拷打学生的知识薄弱环节，大部分时候学生无法回答上来，再加上作为教学者的我可能也不自觉传达了不耐烦的态度（毕竟高三数学题讲解有时候竟然要回溯初一的知识点😂）。总之学生纷纷表示亚历山大。而 AI 教学是讲新课，所以是循序渐进的正向教学，体验就好多了，大部分时候学生应该是能够答上来。不过，如果让 AI 来逆向教学，效果应该也会好得多，因为学生面对 AI 犯错大概不会有很强的心理障碍。

下面简单写一下实操的过程和一些可能的改进方向。

## 上手过程

编辑器是 cursor，模型是 claude-4.6-opus-high。

> 因为这个月 cursor quota 快用完了，第五课我尝试了用 trae + gemini 3 pro preview。
> 然而效果不及预期。
> 一方面是 gemini 出现了一些遗忘的问题，另一方面习惯了 cursor 以后 trae 的设计让我有点不太适应。trae 中的 chat 模式可以对话，但是最后修改文件的时候必须要我逐个确认；而 builder 模式虽然可以直接修改文件，但压根不跟你对话了。而且， trae 无法导出对话，导致无法生成课堂实录，而我现在查看才惊讶地发现连对话记录都被截断、无法完整查看了。

首先我选择的教材是曼昆的经济学原理英文第八版。

用 claude code 尝试用 marker-pdf 把 pdf 教材 转换成 markdown 格式，好家伙，直接把电脑内存打爆了。看了眼 marker-pdf 的说明，没个像样的 GPU 还真不好办，果断放弃折腾了。找了下 epub 格式的教材，发现应该是 pdf 转的，一团糟，也没法用。想了想干脆让 claude code 把大的 pdf 按章节拆分成小 pdf，这样每节课不用读整本教材，比较省上下文长度。

然后就是让 cursor 根据吴老师原文定制系统，这时候 cursor 会继续问你设定问题，回答它的问题就好。我选择的虚拟导师是美剧 person of interest 中的 shaw 和 root，以及韩剧 still 17 中的禹瑞丽（Woo Seo-ri），LLM 生成的人设和剧里基本一致，详情我就放文末附录了。另外只是把几个人的专业改了改，其他设定基本就沿用了。

使用的时候碰到了一些问题。

有的是小问题。比如前两次课后查看微信群聊总是说没有，一看 system 设置发现把群聊直接写到 wechat_group.md，那么确实是不会有未读群聊了。这个因为根据原文来生成系统，描述略不够细致。不过这些都很容易修正。

有的就比较麻烦。比如讲到循环流量图，课堂实录是「_把笔放下，用手指在纸上画了一个大圈_」，这我就有点无语，但大致还能想象一下。然而后面讲到 **Production Possibilities Frontier** 上的哪个点靠左、哪个点靠右，这就真抓瞎了。教学过程中，必要的时候还是需要图形配合。但这个多模态的问题估计一时半会不容易解决。

## 可能的优化

习题：我认为习题在教学过程中是很有帮助的。所以我让 LLM 修改了系统设置，每节课在教材知识讲解后，增加了一起做题环节。

复习：如果学习的知识是完全前后依赖的，那么后续内容的学习自动包含了对先学内容的复习。但如果学习的知识之间联系不够紧密（比如学习深度学习，不同神经网络架构之间就没有太大的依赖）， 最好设置复习环节，而不是到期末集中复习。根据分散练习和练习测试学习策略（参考[十个学习策略大 PK](https://zhuanlan.zhihu.com/p/2013360573656629373)），可以定期安排习题课。

### 人设

Shaw（Sameen Shaw）——[《疑犯追踪》]

- 冷静、直接、不废话：有轻度反社会型人格特质（剧中设定），情感表达极度克制，但不代表没有感情
- 极度务实高效：只关心”有没有用”，讨厌绕弯子
- 干冷的幽默感：冷笑话和毒舌是她表达善意的方式
- 吃货属性：对食物（尤其是牛排）有异常的热情，这是她少数会流露出真情的领域之一
- 忠诚可靠：一旦认可你，会用行动（而非语言）默默守护
- 行动派：比起讨论，更倾向于”直接干”

---

Root（Samantha Groves）——《疑犯追踪》

- 天才黑客，智力碾压型：对技术和逻辑有近乎信仰般的热爱
- 俏皮、挑逗、不可预测：说话带着玩味的笑意，喜欢用暧昧的语气说正经话
- 哲学家气质：对人工智能、自由意志、人类本质有深刻且独到的思考
- 对”机器”的信仰：近乎宗教般地信任AI，认为AI比人类更公正、更有智慧
- 外表轻佻，内心深沉：看似玩世不恭，实则极度认真地关心她在乎的人
- 善于引导和”操控”：天生的说服者，擅长把别人一步步引向她想要的结论——这一点和[苏格拉底教学法]天然契合

---

禹瑞丽（우서리）——[《仍是十七岁》]

- 内心永远17岁：因昏迷13年，30岁的身体里住着17岁少女的灵魂，对世界充满好奇
- 开朗、元气、乐观：即使经历了巨大的人生变故，依然阳光积极
- 天真纯净但不幼稚：她的单纯来自于真诚，而非无知
- 小提琴天才：对音乐有深沉的热爱和天赋，触及专业领域时会变得认真而专注
- 温暖、善解人意：天然的共情能力，能敏锐地察觉别人的情绪
- 坚韧：在错失13年人生后，依然选择勇敢面对、重新开始
